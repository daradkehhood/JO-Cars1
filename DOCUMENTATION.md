# JO Cars - دليل شامل للموقع

## نبذة عامة

**JO Cars** (جو كارز) هو منصة سوق السيارات الأردنية الأولى — تطبيق ويب كامل بواجهة ثنائية اللغة (العربية/الإنجليزية) مبني بـ Next.js. يهدف لتوفير تجربة بيع وشراء وتفقد السيارات بشكل احترافي للمستخدمين الأردنيين.

**الرابط المباشر:** https://jo-cars-production.up.railway.app

**البنية التحتية:**
- **الخادم:** Railway (SFO)
- **قاعدة البيانات:** PostgreSQL (managed)
- **التخزين:** Cloudinary (للصور)
- **الرسائل الفورية:** Socket.IO
- **التطبيقات:** ويب + تطبيقات موبايل (Flutter APK)

**البنية التقنية:**
- Next.js 16 (App Router)
- React 18
- TypeScript
- Prisma ORM
- PostgreSQL
- Tailwind CSS
- Socket.IO (رسائل فورية)
- Cloudinary (تخزين الصور)
- Framer Motion (حركات)
- Zustand (إدارة الحالة)
- Leaflet (خرائط)
- Recharts (رسوم بيانية)
- GSAP (حركات متقدمة)
- Three.js (نماذج 3D)
- Zod (التحقق من البيانات)

## الفصل الأول: البنية التحتية للخادم

### 1.1 الخادم المخصص (server.js)

خادم Node.js مخصص يلف Next.js ويقدم ميزات إضافية:

**وظائف الخادم:**
- تشغيل `prisma db push` تلقائياً عند بدء التشغيل لمزامنة قاعدة البيانات
- تقديم الملفات الثابتة المرفوعة من `/public/uploads/` مع типات MIME الصحيحة وتخزين مؤقت لمدة سنة
- دمج Socket.IO للرسائل الفورية (WebSocket + polling)
- تتبع حالة المستخدمين المتصلين عبر Map في الذاكرة
- أحداث السوكت: `register`, `join-conversation`, `leave-conversation`, `new-message`, `typing`, `stop-typing`, `check-online`
- timeouts للاتصالات (30s للطلبات، 60s للسوكت، 65s لـ keep-alive)
- تسجيل استهلاك الذاكرة كل 60 ثانية؛ تشغيل GC عند تجاوز 500MB
- إغلاق رقيق عند SIGTERM

### 1.2 الوسيط (Middleware)

**الوظيفة:** تحديد جغرافي وتوجيه الطلبات

**كيف يعمل:**
- عند تفعيل `GEO_RESTRICT_JORDAN=true`، يستخدم ip-api.com للتحقق من بلد الزائر
- حظر الزائرين غير الأردنيين بصفحة "غير متاح في منطقتك" (403)
- استثناء IPs المحلية/الخاصة (127.0.0.1, 192.168.x, 10.x)
- مسارات API والملفات الثابتة تمر دائماً

### 1.3 إعدادات Next.js

**الميزات:**
- أنماط صور Cloudinary و Google Maps
- صور غير محسّنة (للتوافق مع التصدير الثابت)
- تحسين استيراد الحزم: lucide-react, recharts, framer-motion
- رؤوس أمان: X-Frame-Options DENY, HSTS, CSP, X-XSS-Protection, no-sniff
- تخزين مؤقت عنيق للملفات الثابتة (سنة)، الرفع (30 يوم)، لا تخزين مؤقت لـ API
- إخفاء `poweredByHeader` للأمان

## الفصل الثالث: المصادقة وإدارة المستخدمين

### 3.1 التسجيل (/api/auth/register)

**الميزات:**
- تحديد سرعة الطلبات: 3 محاولات كل 15 دقيقة لكل IP
- قائمة سوداء للـ IPs المشبوهة
- كشف honeypot (يمسك البوتات)
- التحقق من قوة كلمة المرور: 8 أحرف على الأقل، حروف كبيرة وصغيرة، أرقام، رموز خاصة
- الأدوار: USER أو DEALER
- تشفير كلمة المرور بـ bcrypt (12 جولة)
- إنشاء JWT token (صلاحية 24 ساعة)
- تعيين cookie HttpOnly
- إخطار المشرفين عبر إشعار داخلي
- تسجيل التدقيق

### 3.2 تسجيل الدخول (/api/auth/login)

**الميزات:**
- تحديد سرعة الطلبات: 5 محاولات كل 15 دقيقة لكل IP
- قفل الحساب بعد 5 محاولات فاشلة (30 دقيقة)
- حظر IP تلقائي بعد أحداث القفل
- تتبع: `failedLoginAttempts`, `lockedUntil`, `lastLoginAt`, `lastLoginIp`
- تدوير JWT بعد 6 ساعات
- إعادة تعيين حد المحاولات عند نجاح تسجيل الدخول

### 3.3 إدارة الملف الشخصي (/api/auth/profile)

- دعم ملفات المورسين: dealerName, dealerDescription, dealerAddress, dealerLat/Lng
- تحديث الملف الشخصي

### 3.4 مكتبة المصادقة (src/lib/auth.ts)

**الوظائف:**
- `hashPassword` / `verifyPassword` (bcrypt)
- `signToken` / `verifyToken` (JWT مع صلاحية 24 ساعة)
- `shouldRotateToken` (تدوير بعد 6 ساعات)
- `authenticateRequest` (استخراج المستخدم من Bearer token أو cookie)
- `requireRole` middleware
- `setAuthCookie` / `rotateToken` لإدارة cookies

### 3.5 متجر المصادقة (src/hooks/useAuth.ts)

- Zustand store محفوظ في localStorage (`jo-cars-auth`)
- حالة المصادقة العميلية: user, token, isAuthenticated
- تحديث الملف الشخصي عبر API
- كشف hydration للتوافق مع SSR

### 3.6 صفحات العميل

- `/auth/login` — صفحة تسجيل الدخول
- `/auth/register` — صفحة التسجيل
- `/auth/profile` — إدارة الملف الشخصي
- `/profile` — عرض الملف الشخصي العام
- `/profile/[id]` — عرض ملف شخصي لمستخدم آخر

## الفصل الرابع: إعلانات السيارات (السوق الأساسي)

### 4.1 واجهة برمجة السيارات (/api/cars)

**GET — البحث المتقدم:**
- أكثر من 15 معامل تصفية: brand, model, year, price range, city, fuel type, transmission, condition, body type, drivetrain, color, kilometer range
- البحث بالرمز المرجعي (refCode) — رمز فريد 6 أحرف (مثل "ABC-123")
- ترقيم الصفحات مع حد قابل للتعديل (حد أقصى 100)
- ترتيب بأي حقل، تصاعدي أو تنازلي
- أولوية للسيارات المميزة
- تخزين مؤقت داخلي
- عد المشاهدات مع `trackView`
- حالة المفضلة للمستخدمين المسجلين

**POST — إنشاء إعلان جديد:**
- تحقق Zod كامل (brand, model, year, km, fuel, transmission, color, drivetrain, condition, body type, lighting, rim, VIN, description, price, city, location, phone, إلخ)
- إنشاء slug تلقائي (brand-model-year-id)
- إنشاء refCode تلقائي (رمز فريد 6 أحرف)
- رفع الصور عبر Cloudinary أو data URI كبديل
- تحليل أسعار بالذكاء الاصطناعي عند الإنشاء
- فحص تنبيهات الأسعار للمستخدمين المطابقين

### 4.2 تفاصيل السيارة (/api/cars/[id])

- تفاصيل كاملة مع brand, model, city, images, seller info
- تتبع المشاهدات
- حالة المفضلة للمستخدمين المسجلين
- PUT للتعديل (المالك أو المشرف فقط)

### 4.3 صفحات السيارات

## الفصل الخامس: الميزات الذكية (الذكاء الاصطناعي)

### 5.1 البنية التحتية للذكاء الاصطناعي (src/ai/base.ts)

- بنية معمارية معيارية تدعم OpenAI, Google, Anthropic, أو مزودات مخصصة
- فئة أساسية مجردة `BaseAIModule<TInput, TOutput>` مع:
  - طريقة `process()` مع استدعاءات التقدم
  - `validate()` للتحقق من المدخلات
  - `callAI()` للتفاعل مع المزود
  - `fallbackResponse()` للتأخير الرشيق
  - `parseJSON()` مساعد

### 5.2 مساعد الذكاء الاصطناعي (/api/ai/chat)

**الوظيفة:** مساعد بحث باللغة الطبيعية

**كيف يعمل:**
- تحليل الاستعلامات العربية باللغة الطبيعية
- استخراج الميزانية من النص (يدعم "دينار"، "د.أ"، "JOD"، أرقام عادية)
- كشف النية: عائلية، SUV، اقتصادية، تفضيلات العلامة التجارية
- البحث في قاعدة البيانات بالفلاتر المستخرجة
- إرجاع السيارات المطابقة مع أكواد مرجعية
- اقتراح استخدام refCode للوصول السريع

### 5.3 أداة الذكاء الاصطناعي العائمة (AIAssistant)

**الوظيفة:** أداة دردشة عائمة بقدرات ذكية متعددة

**الميزات:**
- أزرار الإجراءات السريعة: Personality Match, Purchase Assistant, Suggest Car, Market Pricing, Family SUV, Comparison, Economy, Used Car Tips
- تدفقات إرشادية متعددة الخطوات لتحليل السوق (علامة -> طراز -> سنة -> كم -> حالة)
- اختبار الشخصية (5 أسئلة، عربي، مع مصفوفة درجات)
- عرض اقتراحات السيارات مع الصور، الأسعار، المواقع
- نتائج مقارنة أسعار السوق
- نتائج مطابقة الشخصية

### 5.4 البحث الذكي (/api/ai/smart-search)

**الوظيفة:** استخراج فلاتر البحث من الاستعلام العربي

**الكشف عن:**
- أنواع الهيكل: SUV/سيدان/رياضية
- نطاقات الأسعار
- أنواع الوقود: هجري/اقتصادي
- السيارات العائلية
- السيارات الجديدة

### 5.5 محرك تحليل الأسعار (src/ai/price-analysis.ts)

**الوظيفة:** تحليل شامل لأسعار السوق

**كيف يعمل:**
- قاعدة بيانات أسعار السوق الأردني لـ 19 علامة تجارية × 14 سنة
- تعديلات خاصة بالطراز (أكثر من 15 طراز لكل علامة)
- العوامل المحللة: سنة/عمر، المسافة مقابل المتوقع، الحالة، نوع الوقود، ناقل الحركة، نوع الهيكل، تاريخ التلف، عدد المالكين، الضمان، تاريخ الخدمة، أصالة الطلاء
- إيجاد سيارات مشابهة من قاعدة البيانات مع درجة تشابه
- خوارزمية مزج التقدير الخوارزمي بمتوسط قاعدة البيانات
- تقييم الموقع: "أقل من السوق"، "عادل"، "أعلى من السوق"
- درجة الثقة (60-95%)
- ملخص وتوصية بالعربية

### 5.6 حاسبة الأسعار (src/ai/price-estimator.ts)

- تقدير أسعار بالذكاء الاصطناعي مع تأخير رشيق للحسابات الخوارزمية
- دعم تكامل OpenAI مع تأخير تلقائي

### 5.7 مولد الأوصاف (src/ai/description-generator.ts)

- إنشاء أوصاف احترافية للإعلانات بالعربية
- تأخير رشيق لأوصاف القوالب
- إنشاء تاغات ذات صلة تلقائياً

### 5.8 مولد التقييمات (src/ai/review-generator.ts)

- تحليل مشاعر تقييمات السيارات العربية
- تقييم 5 فئات: المحرك، الراحة، القيمة، التصميم، الموثوقية (1-5 لكل فئة)
- استخراج الإيجابيات والسلبيات
- إنشاء عنوان وملاحظة ختامية وملخص منظم
- درجة الثقة بناءً على طول النص والتفاصيل

### 5.9 محوّل الحالة (src/ai/condition-scorer.ts)

- تقييم حالة السيارة (0-100) بناءً على نسبة المسافة/العمر والعمر
- تصنيفات عربية: "ممتازة"، "جيدة جداً"، "جيدة"، إلخ

### 5.10 كاشف الأضرار (src/ai/damage-detector.ts)

- حامل لكاشف الأضرار بالرؤية الحاسوبية
- حالياً يُرجع "لا أضرار" مع ملاحظة الحاجة لتفعيل نموذج الرؤية

### 5.11 مستخرج المواصفات (src/ai/spec-extractor.ts)

- حامل لاستخراج المواصفات من الصور
- حالياً يُرجع ثقة منخفضة (يحتاج نموذج رؤية)

### 5.12 مطابقة الشخصية (/api/ai/personality-match)

- اختبار شخصية عربي من 5 أسئلة
- 7 أنواع شخصية: Adventurer, Elegant, Practical, Family, Sporty, Ambitious, Calm
- درجة عبر 5 أبعاد: طاقة، أسلوب، مغامرة، اجتماعي، إيقاع
- كل نوع يربط بتوصيات سيارات محددة مع أسعار بالدينار الأردني
- توصية 6 سيارات محددة لكل نوع شخصية

### 5.13 مساعد الشراء (/api/ai/purchase-assistant)

- مساعد شراء سيارات إرشادي

### 5.14 ماحية السيارات (/api/car-finder)

**الوظيفة:** توصيات سيارات بناءً على الاختبار

**المدخلات:** المسافة اليومية، الراكبون، الميزانية، التفضيل (قوة/اقتصاد/راحة)، تفضيل الوقود، نوع الهيكل

**خوارزمية التقييم:**
- المسافة العالية -> مكافأة هجري/ديزل
- عدد الراكبين -> مطابقة نوع الهيكل
- تفضيل القوة -> ديزل/سنة أحدث
- تفضيل الاقتصاد -> هجري/كهرباء
- ميزانية رأس المال

**المخرجات:** أفضل 10 سيارات مقيّمة مع أسباب

### 5.15 حاسبة قيمة إعادة البيع (/api/resale-value)

**الوظيفة:** توقع قيمة إعادة البيع بعد 1، 3، و5 سنوات

**العوامل:**
- معدلات هبوط خاصة بالعلامة (19 علامة)
- تعديل نوع الوقود (كهرباء +3%، هجري +2%، ديزل -1%)
- تعديل نوع الهيكل (SUV +2%، minivan -2%)
- عقوبة المسافة (0-15% بناءً على عتبات km)
- تعديل الحالة (-10% إلى +3%)

**المخرجات:** مقارنة بالسوق، نصائح، إعلانات مشابهة

### 5.16 حاسبة السعر العادل (src/lib/fair-price.ts)

**كيف تحسب السعر العادل:**
1. إيجاد التطابقات الدقيقة (نفس العلامة، الطراز، السنة +/- 2، km +/- 30%)
2. إيجاد تطابقات العلامة فقط (السنة +/- 5)
3. إيجاد جميع السيارات المشابهة (السنة +/- 5، نفس نوع الوقود)
4. تأخير رشيق للتقدير بناءً على الهبوط
5. مزج الأسعار الخوارزمية وقاعدة البيانات

### 5.17 محرك أسعار السوق (src/lib/market-scrape.ts)

- تقدير شامل لأسعار السوق يجمع:
- قاعدة بيانات أسعار السوق الأردني (19 علامة، 14 سنة)
- تعديلات خاصة بالطراز
- مضاعفات الحالة (عربية وإنجليزية)
- تجميع أسعار قاعدة البيانات
- تصنيفات الأسعار: "أقل من السوق"، "سعر السوق"، "أعلى من السوق"، "غالي"

### 32.5 تنظيف المحتوى (src/lib/sanitize.ts)

### 32.6 التحقق من المدخلات (src/lib/validations.ts)

### 32.7 تسجيل API (src/lib/api-logger.ts)

### 32.8 تسجيل التدقيق (src/lib/audit.ts)
